CrimHub

January - February 2015

The tools available for course selection at Carleton University can be large and unwieldy. Often students will find themselves using pen and paper to write out their timetables before course registration. CrimHub is a solution to the problem of course selection for Criminology students, a web application which allows them to filter by required courses, drag those courses into a timetable, and see if they are on track for graduation.

The design for CrimHub was developed as part of a team for the course COMP 3008 (Human-Computer Interaction). This project explored the design life cycle for an application through gathering client requirements, wireframing various designs, and performing user testing.

UX, Requirements Gathering, User Testing, Design Lifecycle, Balsamiq, Team Project, Carleton University

cuTPS

September - December 2014

The Carleton University Textbook Publishing System addresses the issue of providing course material to students at a lower cost by allowing them to purchase sections of their textbooks as electronic copies online. This demo project was developed for the course COMP 3004 (Object Oriented Software Engineering).

As a team, we collected client requirements, designed a system using recognized software design patterns, documented and developed a prototype. I was responsible for designing and creating model objects to represent the real-world objects in our system.

C++, Qt, Software Lifecycle, Team Project, Carleton University

Tic Tac Toe

January - April 2014

Tic Tac Toe is a popular, simple game in which players take turns marking spaces in a 3x3 grid. In the course COMP 2601 (Mobile Applications), I developed a version of Tic Tac Toe for Android devices, and a version for iOS devices. The purpose of the project was to demonstrate an understanding of the Android and iOS SDKs by creating a functional application.

Android SDK, iOS SDK, Individual, Carleton University
